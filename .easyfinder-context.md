# EasyFinder — AI Dev Context (Source of Truth)

## Repo + Branch
- Repo: EasyFinder (monorepo)
- Mainline branch: app
- Local OS: Windows
- Shell: PowerShell
- Editor: VS Code available, primary workflow is PowerShell + Codex PRs

## Monorepo Architecture
- Monorepo tool: Turborepo (turbo.json)
- Package manager: pnpm 9.6.0
- Workspaces: apps/*, packages/*

### Packages
- apps/api — Backend API (Fastify, MongoDB, JWT)
- apps/web — Frontend SPA (React + Vite + TypeScript + Tailwind + React Query)
- packages/shared — Shared types + demo data + scoring logic (@easyfinderai/shared)

## Deployment Targets
- Backend API: Fly.io
  - Dockerfile-based build
  - Runtime: apps/api
- Frontend Web: Vercel
  - Project root: apps/web
  - SPA rewrites handled via vercel.json

## CI/CD
- Workflow: .github/workflows/ci.yml
- Triggers: push on branch app + all PRs
- Steps: pnpm install --frozen-lockfile, pnpm -w typecheck, pnpm -w build
- CI blocks committing:
  - **/.env
  - **/.env.*
  - **/node_modules/**
  - **/dist/**
  - **/.turbo/**

## Codex Workflow Rules
- Codex creates small, scoped PRs
- Fernando reviews PR diff on GitHub
- PR merges into branch app
- Local machine runs git pull after merge
- All commands executed via PowerShell
- Changes must be minimal, deterministic, and production-safe

## Guardrails (Non-Negotiable)
Codex must NEVER modify:
- **/node_modules/**
- **/dist/**
- **/.turbo/**
- .git/**
- **/.env
- **/.env.*

Codex must avoid:
- Root config refactors unless explicitly requested
- Repo-wide formatting changes
- Dependency upgrades unless explicitly requested

## Environment Policy
- .env files are local-only
- Only .env.example may be committed
- apps/api/.env was previously tracked and is now blocked via CI

## Current Constraints
- One instruction at a time
- PowerShell-compatible commands
- Always verify with command output before proceeding

## Known Quirks
- apps/web TypeScript may treat ScoringConfig.weights as possibly undefined due to workspace boundary typing.
  - Mitigation: explicit casting at usage site.
- GitHub may show repository moved warning due to casing change; treat as informational.

## Definition of Done for PRs
- Scope limited to requested files
- No forbidden paths modified
- pnpm -w typecheck passes
- pnpm -w build passes
- PR summary includes what changed and why
